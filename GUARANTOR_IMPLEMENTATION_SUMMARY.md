# Реализация функционала "Гарант" - Краткое резюме

## ✅ Что реализовано

### 🗄️ Модели базы данных
- **GuarantorRequest** - заявки на гарантов
- **Guarantor** - активные отношения гарант-клиент  
- **ContractFile** - файлы договоров
- Обновлена модель **User** с новыми связями

### 📡 API эндпоинты

#### Основные функции гарантов:
- `GET /guarantor/info` - информация о функции (для кнопки "?")
- `POST /guarantor/request` - создание заявки на гаранта
- `POST /guarantor/respond/{request_id}` - ответ на заявку
- `GET /guarantor/my-info` - информация о гарантах пользователя
- `POST /guarantor/sign-contract` - подписание договоров
- `POST /guarantor/check-user-eligibility` - проверка платежеспособности
- `GET /guarantor/contracts` - получение договоров

#### Административные функции:
- `POST /admin/approve-user` - одобрение/отклонение с предложением гаранта
- `GET /admin/pending-users` - пользователи ожидающие одобрения
- `POST /guarantor/upload-contract/{type}` - загрузка договоров

### 📱 SMS интеграция
- Приглашение гарантов: "ФИО выбрал(а) вас в качестве Гаранта. Перейдите по ссылке и скачайте приложение"
- Уведомление об отказе с предложением гаранта
- Тестовый режим с кодом "6666"

### 📄 Система договоров
- Загрузка и хранение договоров гаранта и субаренды
- Последовательное подписание (сначала договор гаранта, затем субаренды)
- Файловое хранение в `uploads/contracts/`

## 🔄 Логика работы

### 1. При отказе в регистрации
```
Администратор → Отклоняет пользователя → SMS с предложением гаранта
```

### 2. Добавление гаранта
```
Пользователь → Указывает данные гаранта → 
├─ Гарант не найден → SMS приглашение
└─ Гарант найден → Создание заявки → Проверка платежеспособности
```

### 3. Принятие заявки
```
Гарант → Принимает заявку → Подписывает договор гаранта → Подписывает договор субаренды
```

## 🧪 Тестирование

### Локальное тестирование:
1. Установить `SMS_TOKEN=6666` для тестового режима
2. Запустить приложение: `uvicorn main:app --reload`
3. Открыть Swagger: `http://localhost:8000/docs`
4. Использовать код "6666" для быстрой авторизации

### Основные тест-кейсы:
- ✅ Создание заявки на незарегистрированного гаранта (SMS отправка)

- ✅ Принятие/отклонение заявки гарантом
- ✅ Проверка платежеспособности пользователя
- ✅ Подписание договоров в правильной последовательности
- ✅ Отклонение пользователя с предложением гаранта

### Безопасность:
